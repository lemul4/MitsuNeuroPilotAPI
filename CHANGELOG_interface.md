# Changelog: i-MiEV Auto Control System

---

## [Архитектурный анализ] — 17.04.2026 – 23.04.2026

### Выявленные проблемы
* **Монолитная структура**: логика управления, обработка CAN-шины и отрисовка интерфейса находились в одном пространстве имен, что усложняло масштабирование.
* **Синхронный ввод-вывод**: чтение данных из Serial-порта блокировало основной поток GUI, снижая FPS интерфейса при интенсивном обмене данными.
* **Отсутствие изоляции ИИ**: интеграция модулей автопилота приводила к нестабильности системы из-за отсутствия выделенных потоков для тяжелых вычислений.
* **Трудности отладки**: прямая зависимость виджетов от аппаратных функций мешала полноценному использованию режима `VIRTUAL_DEMO_MODE`.

### Сформулированные требования
* Переход на MVC: `VehicleState`, `MainWindow`, `AppController`.
* Взаимодействие через сигналы PySide6.
* Изоляция ИИ-агента, телеметрии и аппаратного обмена в отдельных потоках.
* Возможность запуска интерфейса без подключенного автомобиля.

## [1.1.0-refactor] — 2026-04-29

### Added
* **Модуль `AppController`**: добавлен центральный диспетчер, координирующий виртуальную физику, аппаратный слой и UI.
* **Виджет AI Vision**: реализована панель отображения видеопотока с динамическим масштабированием кадра.
* **Асинхронный `LeadAgentThread`**: добавлен запуск ИИ-моделей в отдельном потоке.
* **`RawTelemetryJsonlReader`**: добавлен модуль чтения телеметрии из JSONL-файлов для синхронизации симуляции с интерфейсом.

### Changed
* **Реорганизация UI**: главное окно разделено на зоны `Контроль`, `Настройки`, `Аналитика`.
* **Опрос клавиатуры**: управление переведено на таймер `key_poll_timer` с интервалом 50 ms.
* **Изоляция команд управления**: аппаратные команды `Serial_Data` перенесены внутрь контроллера.

### Fixed
* **Input Lag**: устранена задержка реакции руля при зажатых клавишах управления.
* **Focus Loss**: исправлена потеря фокуса клавиатуры после взаимодействия со спинбоксами PID.

### Improved
* **Модульность UI**: `SteeringWidget` выделен в независимый компонент.
* **Графики**: обновление `pyqtgraph` централизовано через `update_dashboard`.

## [1.2.0-stabilization] — 2026-05-07

### Added
* **Автоматическое переподключение**: добавлен механизм повторного подключения при потере связи с аппаратной частью.
* **Контроль состояния соединения**: добавлена проверка доступности Serial/CAN-канала перед отправкой команд.
* **Безопасный режим управления**: при потере связи система переводит управление в безопасное состояние.
* **Расширенная диагностика**: добавлены сообщения о статусе подключения, ошибках обмена и состоянии активных потоков.
* **Логика обработки кнопок**: добавлены отдельные обработчики для основных управляющих действий интерфейса.

### Changed
* **Управление кнопками**: логика UI-кнопок перенесена из представления в `AppController`.
* **Обработка аппаратных команд**: команды теперь проходят через единый слой валидации перед отправкой.
* **Поведение виртуального режима**: `VIRTUAL_DEMO_MODE` больше не зависит от наличия подключенного Serial-устройства.
* **Обновление состояния автомобиля**: данные из симуляции, телеметрии и аппаратного слоя синхронизируются через общую модель `VehicleState`.
* **Обработка ошибок**: критические ошибки больше не блокируют основной поток интерфейса.

### Fixed
* Исправлена некорректная работа кнопок при повторном нажатии.
* Исправлены случаи зависания интерфейса при потере связи с аппаратной частью.
* Исправлены конфликты между виртуальным управлением и аппаратными командами.
* Исправлена нестабильность при одновременном обновлении графиков, телеметрии и AI Vision.
* Исправлены ошибки повторной инициализации потоков после остановки или перезапуска системы.

### Improved
* Повышена стабильность главного окна при длительной работе.
* Оптимизировано обновление dashboard-панели.
* Снижена нагрузка на CPU при работе графиков и фоновых потоков.
* Улучшена читаемость структуры проекта.
* Упрощена дальнейшая интеграция новых модулей автопилота.
* Улучшена отказоустойчивость при работе без подключенного автомобиля.
