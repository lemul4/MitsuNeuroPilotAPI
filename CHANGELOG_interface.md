# Changelog: i-MiEV Auto Control System
---
## [Архитектурный анализ] — 17.04.2026 – 23.04.2026

### Выявленные проблемы
*   **Монолитная структура**: догика управления, обработка CAN-шины и отрисовка интерфейса находились в одном пространстве имен, что вызывало трудности при масштабировании.
*   **Синхронный ввод-вывод**: чтение данных из Serial-порта блокировало основной поток GUI, снижая FPS интерфейса при интенсивном обмене данными.
*   **Отсутствие изоляции ИИ**: интеграция модулей автопилота приводила к нестабильности системы из-за отсутствия выделенных потоков для тяжелых вычислений.
*   **Трудности отладки**: прямая зависимость виджетов от аппаратных функций исключала возможность полноценного использования режима `VIRTUAL_DEMO_MODE`.

### Сформулированные требования к новой структуре
*   **Переход на MVC**: четкое разделение на Модель (`VehicleState`), Представление (`MainWindow`) и Контроллер (`AppController`).
*   **Событийная модель**: взаимодействие через систему сигналов PySide6 для полной развязки модулей.
*   **Потоковая изоляция**: вынос ИИ-агента и записи телеметрии в независимые фоновые процессы.

## [1.1.0-refactor] — 2026-04-29 (Текущий статус)

### Added
*   **Модуль управления `AppController`**: внедрен центральный диспетчер, координирующий потоки данных между виртуальной физикой, реальным авто и UI.
*   **Виджет AI Vision**: реализована панель отображения видеопотока с поддержкой динамического масштабирования кадра.
*   **Асинхронный агент `LeadAgentThread`**: добавлена поддержка запуска ИИ-моделей в отдельном потоке с неблокирующим логированием.
*   **Система чтения телеметрии**: добавлен `RawTelemetryJsonlReader` для синхронизации состояния симуляции с интерфейсом в реальном времени.

### Changed
*   **Реорганизация UI**: основное окно разделено на три функциональные зоны (Контроль, Настройки, Аналитика).
*   **Оптимизация опроса клавиатуры**: управление переведено на таймерный опрос `key_poll_timer` (50ms), что обеспечило плавность хода виртуальной модели.
*   **Изоляция управления**: аппаратные команды (`Serial_Data`) теперь зашиты внутри контроллера, а не вызываются из кнопок напрямую.

### Fixed
*   **Input Lag**: Устранена задержка реакции руля при зажатых клавишах управления за счет накопления состояния в `pressed_keys`[cite: 12, 14].
*   **Focus Loss**: Исправлена проблема потери фокуса клавиатуры после нажатия на спинбоксы настроек PID (установлена политика `NoFocus` для сервисных виджетов)[cite: 14].

### Improved
*   **Модульность**: виджет руля `SteeringWidget` выделен в независимый компонент, не зависящий от глобального состояния приложения.
*   **Визуализация**: графики `pyqtgraph` теперь обновляются централизованно через метод `update_dashboard`, что снизило нагрузку на CPU.
