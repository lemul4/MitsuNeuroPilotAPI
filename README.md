
# MitsuNeuroPilot — интеллектуальный автопилот в среде CARLA

## Общее описание проекта
**MitsuNeuroPilot** — это исследовательский и прикладной программный продукт, предназначенный для разработки, обучения и оценки интеллектуальных агентов автономного вождения в симуляторе **CARLA**. Проект ориентирован на моделирование реальных дорожных сценариев и построение автопилота, способного безопасно и устойчиво управлять автомобилем без участия человека.

Система сочетает классические алгоритмы управления, методы машинного обучения и нейросетевые подходы, а также поддерживает микросервисную архитектуру для онлайн-взаимодействия с агентом.

Проект разрабатывался как:
- исследовательская платформа;
- основа для дипломной / научной работы;
- экспериментальная среда для тестирования архитектур автопилота.

---

## Цели и задачи
Цель проекта — разработка интеллектуального агента автопилота, способного:
- безопасно передвигаться в сложной дорожной обстановке;
- следовать маршрутам различной протяжённости;
- взаимодействовать с другими участниками движения;
- собирать датасеты для обучения нейросетей;
- работать как автономный агент и как сетевой микросервис.

Для достижения цели были поставлены следующие задачи:
- интеграция с симулятором CARLA;
- разработка архитектуры автопилота (чистая архитектура + микросервисы);
- реализация нескольких типов агентов (PID, имитационный, микросервисный);
- организация системы сбора и сохранения данных;
- реализация системы оценки (leaderboard, сценарии, метрики);
- обеспечение воспроизводимости экспериментов.

---

## Архитектура проекта
Архитектура проекта является **гибридом чистой архитектуры, многослойного подхода и микросервисов**.

### Ключевые принципы:
- разделение доменной логики и инфраструктуры;
- независимость бизнес-логики от CARLA;
- возможность запуска агента:
  - напрямую через evaluator,
  - либо как отдельного FastAPI-микросервиса;
- слабая связанность компонентов;
- масштабируемость и расширяемость.

### Основные архитектурные уровни:
- **Domain** — бизнес-логика автопилота, политики управления, модели принятия решений;
- **Use Cases** — сценарии использования (оценка, обучение, тестирование);
- **Adapters** — агенты, сенсоры, интерфейсы CARLA;
- **Services** — микросервисы (автопилот, оценка, обучение);
- **Infrastructure** — CARLA, сценарии, карты, сторонние библиотеки.

## Типы агентов
В проекте реализованы несколько вариантов агентов:

### 1. Autopilot Service Agent
Классический агент, используемый для:
- сбора данных;
- управления автомобилем на основе правил и PID-контроллеров;
- генерации обучающих выборок.

### 2. Imitation Learning Agent
Нейросетевой агент, который:
- использует обученную модель PyTorch;
- принимает решения на основе состояния среды;
- проходит сценарии leaderboard.

### 3. Autopilot Microservice
Микросервис на базе **FastAPI**, который:
- поднимается как отдельное приложение;
- управляет автомобилем через HTTP-запросы;
- позволяет тестировать автопилот в интерактивном режиме;
- подходит для интеграции с внешними системами.

## Работа с CARLA
Проект использует:
- **CARLA 0.9.15**
- сценарии ScenarioRunner и Leaderboard;
- пользовательские маршруты и карты;
- синхронный режим симуляции;
- сенсоры: RGB, Depth, Segmentation, GPS и др.

Все взаимодействие с симулятором изолировано в инфраструктурном слое.

## Использование

### 1. Запуск агента с нейросетью
1. Запустите CARLA;
2. Перейдите в `use_cases/evaluation`;
3. В `run_evaluator.ps1` укажите:
```text
TEAM_AGENT=adapters\agents\agents\imitation_service\agent.py
```
4. Запустите:
```powershell
./run_evaluator.ps1
```

### 2. Сбор данных для обучения
1. Запустите CARLA;
2. В `run_evaluator.ps1` укажите:
```text
TEAM_AGENT=adapters\agents\agents\autopilot_service\agent.py
```
3. Данные будут сохранены в папке `dataset`.

### 3. Запуск микросервиса автопилота
```powershell
cd adapters/agents/agents/autopilot_microservice
python -m autopilot_service
```
После запуска:
- откройте http://localhost:8002/docs
- используйте Swagger UI для отправки команд агенту.
