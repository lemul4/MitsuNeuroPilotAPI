# CHANGELOG по индивидуальному заданию на практику

**Проект:** Программная система обучения нейросетевых агентов автономного управления транспортными средствами на базе симулятора CARLA

## 17.04.2026–23.04.2026 — Сбор обучающего датасета на основе прохождения маршрутов экспертом-учителем

**Трудоемкость:** 42 академических часа  
**Цель этапа:** подготовить воспроизводимый пайплайн сбора данных в CARLA, обеспечить запись сенсорных данных и управляющих воздействий экспертного агента, а также первично проверить пригодность полученных маршрутов для дальнейшего обучения модели.

### Добавлено

- Подготовлен и доработан контур запуска экспертного агента для сбора демонстраций в симуляторе CARLA.
- Настроена интеграция экспертного водителя с инфраструктурой Leaderboard/ScenarioRunner.
- Организована запись данных прохождения маршрутов в директорию датасета проекта.
- Добавлена конфигурация экспертного режима сбора данных, включающая:
  - режим сохранения данных для последующего обучения;
  - параметры формата данных `py123d`;
  - отключение нерелевантных для vision-only обучения сенсорных потоков;
  - управление режимами сохранения legacy-выходов;
  - параметры сохранения изображений, управляющих воздействий и служебной информации маршрута.
- Реализована поддержка трехкамерной конфигурации сенсоров для сбора визуальных данных:
  - фронтальная камера;
  - левая фронтальная камера;
  - правая фронтальная камера.
- Настроены параметры камер:
  - разрешение изображений;
  - поле зрения;
  - положение и ориентация камер относительно ego-vehicle;
  - порядок объединения камерных изображений при построении панорамного представления.
- Добавлена логика построения семантической панорамы на основе данных нескольких камер.
- Реализована обработка верхней части изображения при формировании панорамы, чтобы исключить нерелевантные области кадра и привести данные к рабочему формату для обучения.
- Добавлена логика fallback-объединения изображений на случай невозможности построения полной панорамы.
- Расширен набор скриптов для сопровождения датасета:
  - скрипты запуска экспертной оценки;
  - скрипт запуска сбора данных;
  - скрипт подсчета маршрутов;
  - скрипт подсчета количества сэмплов и маршрутов по городам;
  - вспомогательные скрипты для анализа результатов прохождения маршрутов;
  - скрипты удаления ошибочных или выбранных артефактов;
  - скрипты формирования GIF/визуального представления маршрута;
  - скрипты суммаризации времени маршрутов и количества снапшотов.
- Добавлена поддержка запуска CARLA-хоста в разных окружениях:
  - локальный запуск;
  - запуск через WSL;
  - автоматическое определение адреса хоста;
  - fallback-логика при отсутствии явно заданного адреса CARLA.
- Добавлены настройки, повышающие воспроизводимость экспериментов и удобство запуска:
  - переопределение папки маршрутов;
  - настройка endpoint-файлов;
  - настройка путей сохранения результатов;
  - управление параметрами компрессии выходных данных.
- Добавлены настройки Unreal Engine/CARLA для управления ограничениями частоты кадров и снижения влияния графических ограничений на сбор данных.
- Добавлена возможность учитывать и анализировать ошибки прохождения маршрутов, включая ошибки зависания агента, проблемы сценариев и длительность эпизодов.

### Изменено

- Переработан базовый конфиг сенсоров экспертного агента под формат сбора визуальных данных.
- Скорректированы координаты и ориентации камер для получения согласованного набора изображений.
- Обновлен порядок рендеринга камер при построении семантической панорамы.
- Скорректирована логика сохранения данных так, чтобы в датасете оставались только данные, необходимые для последующего обучения модели.
- Обновлены параметры ego-vehicle для прохождения маршрутов в CARLA.
- Настроено использование погодных условий, позволяющее собирать данные не только в одном фиксированном сценарии, а в более разнообразных условиях симуляции.
- Уточнена логика обработки сценариев без дополнительных событий, чтобы такие маршруты могли использоваться в процессе сбора данных.
- Скорректированы параметры определения блокировки транспортного средства:
  - минимальная скорость;
  - максимальное время нахождения в заблокированном состоянии;
  - поведение тестов ActorBlocked/AgentBlocked.
- Доработана логика подсчета статистики прохождения маршрутов.
- Добавлена возможность учитывать нарушения минимальной скорости как диагностический сигнал без обязательного влияния на итоговую оценку маршрута.
- Переработана фильтрация объектов окружения при сборе обучающих данных:
  - транспортные средства;
  - пешеходы;
  - велосипедисты;
  - уязвимые участники дорожного движения.
- Добавлена проверка видимости объектов не только по пикселям камеры, но и по данным LiDAR-точек, использовавшихся как вспомогательный критерий для валидации релевантности объектов в сцене.
- Уточнена логика отбраковки объектов: объект не исключается из выборки только потому, что он слабо представлен в одном источнике данных, если он достаточно представлен в другом источнике.
- Переработан privileged route planner для более корректной обработки маршрутов и объектов в сцене.
- Доработана логика обработки сценариев, чтобы уменьшить количество некорректно завершенных эпизодов и повысить пригодность собранных данных для обучения.

### Исправлено

- Исправлены ошибки запуска сбора данных в разных сетевых конфигурациях.
- Исправлены проблемы с путями к route-файлам.
- Исправлены ошибки сохранения результатов экспертного прохождения.
- Исправлены проблемы с обработкой отдельных сценариев CARLA.
- Исправлены ошибки фильтрации объектов, из-за которых часть релевантных пешеходов, велосипедистов или транспортных средств могла некорректно исключаться из обучающих данных.
- Исправлены проблемы с подсчетом статистики маршрутов и сэмплов.
- Исправлены настройки, связанные с временными ограничениями сценариев и блокировкой агента.
- Уточнена логика сохранения данных так, чтобы исключить избыточные и нерелевантные файлы, но сохранить необходимые изображения, управляющие воздействия и служебные метаданные.

### Полученный результат

- Подготовлен рабочий пайплайн сбора обучающего датасета в CARLA.
- Сформирован набор данных на основе прохождения маршрутов экспертом-учителем.
- Датасет содержит визуальные данные с камер, управляющие воздействия, служебную информацию маршрутов и статистику прохождения.
- Добавлены инструменты контроля полноты датасета: подсчет маршрутов, подсчет сэмплов по городам, анализ длительности маршрутов и обработка ошибочных результатов.
- Данные приведены к формату, пригодному для последующей загрузки в training pipeline и обучения обновленной модели.

## 24.04.2026–7.05.2026 — Переработка TransFuser v6 под vision-only формат с камерными входами

**Трудоемкость:** 54 академических часа  
**Цель этапа:** адаптировать архитектуру TransFuser v6 для обучения нейросетевого агента автономного управления преимущественно по визуальным данным, полученным из камер, с отказом от LiDAR-зависимого входа в рабочем варианте архитектуры.

### Добавлено

- Добавлена новая конфигурация модели для камерного режима обучения.
- Подготовлена отдельная конфигурация модели под формат front-camera perception.
- Добавлен режим backbone, ориентированный на обработку фронтальных камерных изображений.
- Реализованы независимые camera encoder-ветви для обработки изображений с разных фронтальных камер.
- Добавлена stage-wise fusion-логика для объединения признаков камер на разных уровнях encoder-а.
- Добавлен geometry-aware camera-to-BEV projector, преобразующий камерные признаки в BEV-представление.
- Сохранен общий контракт выходов backbone:
  - `bev_features`;
  - `image_features`.
- Обеспечена совместимость новых camera-derived BEV-признаков с существующими head-модулями:
  - planning decoder;
  - CenterNet decoder;
  - BEV semantic decoder;
  - perspective semantic decoder;
  - depth decoder.
- Добавлена поддержка выбора режима сенсоров через параметр конфигурации backbone.
- Добавлена обработка отдельных ключей камерных входов:
  - левая фронтальная камера;
  - правая фронтальная камера;
  - соответствующие intrinsics/extrinsics при их наличии.
- Добавлена поддержка статического forward-пути для ускоренного случая камерной модели без дополнительных динамических входов.
- Добавлена возможность использовать `torch.compile` для оптимизации forward-прохода модели.
- Реализован специализированный forward-путь для камерного режима, в котором модель получает изображения с камер и строит BEV-признаки без LiDAR-ветки.
- Сохранена возможность совместимости с legacy-режимом TransFuser, чтобы архитектура могла работать как в исходном LiDAR+RGB варианте, так и в новом vision-oriented варианте.

### Изменено

- Переработан backbone TransFuser v6:
  - исходный режим `lidar_rgb` сохранен как legacy-вариант;
  - добавлен отдельный камерный режим;
  - признаки BEV в новом режиме формируются из камерных данных;
  - LiDAR-псевдоизображение перестает быть обязательным входом для рабочего vision-only направления.
- Изменен способ подготовки входных данных для forward-прохода:
  - вместо единого RGB-панорамного входа поддерживаются отдельные камерные тензоры;
  - при наличии калибровочных параметров используются intrinsics/extrinsics;
  - при отсутствии динамической калибровки используется ускоренный статический путь.
- Обновлена логика декодирования результатов:
  - perspective semantic head получает image features;
  - depth head получает image features;
  - CenterNet и BEV semantic head получают top-down BEV feature grid;
  - planning decoder может использовать совместимый BEV-контракт.
- Обновлены конфигурационные параметры обучения и модели:
  - режим сенсоров backbone;
  - ключи камер;
  - параметры камерного BEV-проектора;
  - размеры признаковых карт;
  - настройки auxiliary heads;
  - параметры semantic segmentation loss.
- Переработана загрузка и подготовка данных CARLA для соответствия новой архитектуре.
- Добавлена поддержка 14-классовой BEV-сегментации.
- Добавлены новые аугментации для повышения устойчивости модели к вариативности изображений.
- Обновлена логика работы с классами semantic/BEV segmentation, включая дополнительные веса для уязвимых участников движения.
- Перенесен training pipeline на новую архитектуру, чтобы обучение происходило с учетом обновленного формата входов и выходов модели.

### Исправлено

- Исправлены несовпадения между форматом данных, собираемых в CARLA, и форматом данных, ожидаемым моделью.
- Исправлены проблемы миграции training pipeline на новую архитектуру.
- Исправлены ошибки согласования размерностей между backbone, BEV-проектором и decoder-модулями.
- Исправлены ошибки обработки камерных входов при отсутствии части динамической калибровочной информации.
- Исправлены ошибки конфигурации, связанные с выбором режима сенсоров и используемых камер.
- Исправлены проблемы интеграции новых auxiliary losses с основным forward-проходом.
- Исправлены проблемы совместимости новых BEV-признаков с существующими heads модели.

### Технический итог этапа

- TransFuser v6 адаптирован к vision-oriented формату.
- Подготовлена модельная конфигурация, в которой BEV-признаки строятся на основе камерных изображений.
- Реализована камерная ветка backbone с независимыми encoder-ами и последующим объединением признаков.
- Реализован camera-to-BEV projector, позволяющий использовать визуальные признаки в задачах BEV segmentation и object detection.
- Сохранена совместимость с существующими decoder-модулями и training loop.
- Подготовлена модель к обучению на собранном CARLA-датасете.

### Примечание по итоговой архитектуре

В актуальном состоянии кода модельная часть выделяет рабочий камерный режим через front-camera ветви, а трехкамерная конфигурация учитывается на уровне сбора и подготовки визуальных данных. Итоговая логика архитектуры ориентирована на vision-only обучение: ключевые признаки для BEV-представления формируются из изображений, а не из LiDAR-псевдоизображения.

## 7.05.2026–15.05.2026 — Обучение обновленной модели на собранном датасете

**Трудоемкость:** 72 академических часа  
**Цель этапа:** обучить обновленную модель управления транспортным средством на собранных данных, обеспечить контроль процесса обучения, добавить валидацию, ускорить вычисление loss-функций и организовать сохранение контрольных версий модели.

### Добавлено

- Добавлен validation pipeline для контроля качества модели во время обучения.
- Добавлены метрики валидации:
  - суммарный validation loss;
  - unscaled losses;
  - scaled losses;
  - дополнительные metric-значения из модели.
- Добавлена настройка частоты расчета дополнительных validation metrics.
- Добавлена поддержка вычисления validation loss без обновления весов модели.
- Добавлена поддержка сохранения контрольных точек модели по эпохам.
- Добавлено сохранение:
  - весов модели;
  - состояния optimizer;
  - состояния scheduler;
  - состояния gradient scaler при mixed precision;
  - информации о пропущенных gradient steps.
- Добавлены параметры управления хранением checkpoint-файлов:
  - сохранение всех контрольных точек;
  - сохранение выбранных эпох;
  - удаление лишних checkpoint-файлов для экономии места.
- Добавлена возможность сохранения best-модели по метрике качества для совместимых режимов обучения.
- Добавлена поддержка аргумента командной строки для обучения на доле CARLA-датасета.
- Добавлена обработка дробных значений доли датасета в разных локалях.
- Добавлен compiled forward-path через `torch.compile`.
- Добавлена компиляция отдельных loss-функций для ускорения обучения.
- Добавлена focal loss для semantic segmentation.
- Добавлены параметры focal loss:
  - `alpha`;
  - `gamma`;
  - режим включения/выключения;
  - отдельный compiled cache для focal loss.
- Добавлены class weights для повышения значимости малых и уязвимых классов, включая пешеходов и велосипедистов.
- Добавлены вспомогательные оптимизации для ускорения вычисления loss и validation-проходов.

### Изменено

- Обновлен основной training loop под новую архитектуру модели.
- Обновлена логика подготовки batch-данных перед forward-проходом.
- Изменена схема расчета loss:
  - losses агрегируются по источникам данных;
  - losses масштабируются через веса;
  - validation и training используют согласованный механизм расчета;
  - поддерживается раздельное логирование scaled и unscaled components.
- Обновлена логика mixed precision training:
  - используется autocast;
  - сохраняется состояние scaler;
  - учитываются пропущенные gradient steps.
- Оптимизированы validation-проходы:
  - уменьшены избыточные вычисления;
  - добавлена частотность дополнительных метрик;
  - улучшено накопление статистики по validation batch-ам.
- Оптимизированы loss-функции:
  - focal loss реализована без полного materialization one-hot target;
  - добавлена возможность компиляции loss-функций;
  - снижены накладные расходы при работе с сегментационными задачами.
- Обновлена логика cache build root overrides, чтобы training pipeline корректно работал с переопределенными путями датасета.
- Обновлен механизм сохранения модели:
  - модель сохраняется по эпохам;
  - optimizer/scheduler/scaler синхронизированы с checkpoint-ами модели;
  - удаление старых checkpoint-ов выполняется с учетом настроек сохранения.
- Переработан post-train этап, чтобы после обучения корректно сохранялись итоговые артефакты.

### Исправлено

- Исправлены ошибки запуска обучения после миграции на новую архитектуру.
- Исправлены ошибки в training pipeline, связанные с форматами данных и конфигурацией модели.
- Исправлены проблемы validation-прохода.
- Исправлены ошибки расчета loss при работе с новыми segmentation heads.
- Исправлены проблемы производительности при вычислении loss-функций.
- Исправлены ошибки post-train обработки.
- Исправлены проблемы сохранения checkpoint-ов и восстановления состояния обучения.
- Исправлены проблемы работы с dataset cache и переопределением root-путей.

### Полученный результат

- Обновленная модель успешно подготовлена к обучению на собранном CARLA-датасете.
- Реализован полный цикл обучения:
  - загрузка данных;
  - forward-проход новой архитектуры;
  - расчет loss;
  - backpropagation;
  - validation;
  - сохранение checkpoint-ов;
  - постобработка результатов обучения.
- Добавлены механизмы контроля сходимости через validation metrics.
- Добавлены механизмы сравнения промежуточных версий модели через checkpoint-ы.
- Обеспечена возможность выбора рабочей версии модели на основе сохраненных контрольных точек и результатов валидации.
- Training pipeline приведен к состоянию, пригодному для повторного запуска экспериментов и дальнейшего подбора гиперпараметров.
